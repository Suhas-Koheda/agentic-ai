# Screenshot Capture Instructions

Use these instructions to gather the six screenshots required by the assignment from your terminal execution and database query outputs.

---

## 🚀 Step 1: Run the Demonstration Script
Open your terminal, navigate to the `assignment2` folder, and run:
```bash
python demo.py
```

---

## 📸 Step 2: Take Your Screenshots

### 1. Agent Routing Screenshot
*   **What to look for**: The terminal output sections for Query 1, 2, or 3 showing the nodes executing and their classifications.
*   **What to capture**: A screenshot of Query 2 or 3 showing:
    ```text
    ✔ Executed Node: classify_intent
      ├─ Classified Department: Technical Support (or Account)
      ├─ Issue Type: Normal query
      └─ Human Approval Required: No
    ✔ Executed Node: technical_agent
    ```
    This demonstrates the intent classifier node successfully routing the query to the correct department agent.

---

### 2. Human Approval (Interrupt) Screenshot
*   **What to look for**: The execution halting during Query 4.
*   **What to capture**: A screenshot of the terminal when it pauses at:
    ```text
    🛑 INTERRUPT REACHED: Halted at 'human_approval'
    ╭────────────────────── ⚠️ SUPERVISOR APPROVAL REQUIRED ───────────────────────╮
    │ Department: Billing                                                          │
    │ Issue Type: Refund request                                                   │
    ...
    Please enter supervisor action:
      [1] Approve request
      [2] Reject request
    Enter choice (1 or 2): 
    ```
    This proves that LangGraph successfully interrupted execution before the sensitive action node, awaiting supervisor interaction.

---

### 3. RAG Retrieval Screenshot
*   **What to look for**: The detailed responses generated by the agents containing information pulled from local knowledge documents.
*   **What to capture**: The `🌟 Final Output 🌟` panel for Query 1 (retrieved pricing tiers: Standard $10, Professional $29, Enterprise) or Query 3 (retrieved upload size limit: 10MB and file types: .pdf, .png, etc.). This validates that the local BM25 pipeline successfully retrieved the context.

---

### 4. Memory Storage Screenshot
*   **What to look for**: The SQLite database records of the customer interactions.
*   **How to retrieve**: Open a terminal in the `assignment2` folder and query the memory table:
    ```bash
    sqlite3 customer_memory.db "SELECT id, customer_name, department, issue_type FROM customer_interactions;"
    ```
*   **What to capture**: The console output displaying the database rows (including David's refund interaction). This proves that resolved support interaction details are persisted in SQLite.

---

### 5. Memory Recall Screenshot
*   **What to look for**: Query 5 executing, finding the name and retrieving context from the database.
*   **What to capture**: The node execution log at the start of Query 5:
    ```text
    📥 Customer Input: "What was my previous support issue?"
    ⚙️  Workflow Path Execution:
      ✔ Executed Node: load_customer_info_and_history
        ├─ Customer Name: David
        └─ Retrieved Memory History: Found past interaction.
    ```
    This shows the name was recalled from state and used to retrieve past interactions from the database.
